The **Revenue** section connects the engagement data the rest of the Platform produces with the question every publisher eventually asks: which content, audiences, and journeys actually make money? The section is not a billing dashboard. It is a set of analyses that surface the signals correlated with monetization potential.

### What this section covers

| Page | What it covers |
|------|----------------|
| [Revenue Overview](./overview) | The section's purpose and the three workflows it covers |
| [Journey Optimization](./journey-optimization) | How user journeys are modeled and where drop-offs surface |
| [Journey Step](./journey-step) | The sequential position concept that journey analysis is built on |
| [Users Reached Step](./users-reached-step) | The distinct-user denominator for per-step metrics |
| [Intent Progression](./intent-progression) | How the dominant intent shifts across journey steps |
| [Sample Size](./sample-size) | The reliability cut-off for journey data |
| [minConfidence](./min-confidence) | The classification-confidence threshold for intent filters |
| [High-Intent Content](./high-intent-content) | Pages whose audience signals predict conversion readiness |
| [Sponsor Discovery](./sponsor-discovery) | Brand matching against the site's audience and content profile |

### How to read the section

Start with [Revenue Overview](./overview) for the framing. The three workflows — journey optimization, high-intent content, sponsor discovery — can be read in any order depending on which question you're working on. Conversion teams typically start with journey optimization; editorial teams with high-intent content; sales and partnerships teams with sponsor discovery.
